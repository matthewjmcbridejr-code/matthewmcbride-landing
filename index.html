How I Work

Clear specs first Write down what the system should do before writing how. Specs make evaluation possible.
Debug end-to-end Read logs, reproduce locally, isolate the failing component. Not "restart and hope."
Testing and safety checks Inputs validated, outputs checked, destructive actions gated behind explicit confirmation.
Pull request review Changes ship through PRs with diffs a human can actually read. No background commits to main.
Evidence-based evaluation Score outputs against rubrics, log the score, compare versions over time. Vibes are not a measurement.
Practical shipping Real systems running for real users beat perfect systems on a slide. Iterate from there.
